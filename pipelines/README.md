# Data Pipelines - Medallion Architecture

This directory contains the core Spark processing pipelines that transform raw bus waypoints into high-value business insights using the **Medallion Architecture**.

## 1. Medallion Architecture Overview

We organize our data into three distinct layers to ensure reliability, quality, and performance.

### 🥉 Bronze (Raw Layer)
- **Source**: Kafka (`buswaypoint_json` topic).
- **Process**: Spark Structured Streaming.
- **Description**: Captures raw events exactly as they arrive. We add an `ingested_at` timestamp to track the data's journey.
- **Storage**: Apache Iceberg table `bus_bronze.bus_way_point`.

### 🥈 Silver (Refined Layer)
- **Source**: Bronze Layer + Dimension tables.
- **Process**: Batch Processing (Cleaning & Enrichment).
- **Optimizations**:
    - **Deterministic Deduplication**: Switched from simple `dropDuplicates` to **Window Functions** (`row_number()`) ordered by `load_at` to always keep the latest, most accurate record for each vehicle/timestamp pair.
    - **Data Quality**: Implementation of strict filters to remove GPS noise (coordinates at 0,0) and invalid telemetry (unrealistic speeds > 150km/h).
    - **Enrichment**: Joining with route and vehicle metadata to provide business context.
- **Storage**: `bus_silver.bus_way_point`.

### 🥇 Gold (Aggregated Layer)
- **Source**: Silver Layer.
- **Process**: Analytical Aggregations.
- **Outputs**:
    - `trip_summary`: Advanced trip segmentation logic based on terminal proximity and speed.
    - `vehicle_daily_stats`: Summarized performance metrics per day.
    - `gold_bus_dashboard`: Denormalized data optimized for instant Grafana visualization.

## 2. Key Optimizations & Best Practices

To handle the massive scale of the Ho Chi Minh City bus dataset (~34GB in development), several optimizations were implemented:

### ⚡ Partitioning & Data Layout
- **Hidden Partitioning**: Utilized Iceberg's `days(timestamp)` partitioning. This allows users to query by timestamp while Spark automatically prunes partitions at the file level without needing a manual `date` column.
- **Sort Order**: Applied `sortWithinPartitions` on high-cardinality columns like `route_id` and `vehicle`. This clusters similar data together, leading to better min/max metadata and significantly faster data skipping during queries.

### 🛡️ Idempotency & Fault Tolerance
- **Overwrite Partitions**: Used `.overwritePartitions()` instead of `.append()` in batch jobs. This ensures that if a job is re-run for a specific date, it replaces the existing data for that day rather than duplicating it.
- **Metadata Management**: Added `updated_at` to every Silver and Gold table to track when records were last modified.

### 🚀 High-Throughput Ingestion
- **Producer Optimization**: The `json_producer.py` was tuned with increased `linger_ms`, `batch_size`, and `snappy/gzip` compression to achieve ingestion speeds of over **10,000 records per second**.
- **Batch Processing**: Spark flushes data in large, optimized Parquet files to reduce the "small file problem" common in streaming architectures.

### 🧹 Iceberg Table Maintenance
To ensure long-term stability and query performance, we perform regular maintenance tasks:
1.  **Compaction (`rewrite_data_files`)**: Merges the small Parquet files generated by continuous streaming into larger, optimized files. This solves the "Small File Problem" and speeds up reads.
2.  **Rewrite Manifests**: Optimizes the metadata layer by compacting manifest files, improving the speed of plan-time partition pruning.
3.  **Expire Snapshots**: Removes old table versions (snapshots) to reclaim storage space while maintaining a controlled window for "Time Travel".
4.  **Remove Orphan Files**: Deletes physical data files that are no longer referenced by the metadata, ensuring the storage (MinIO) remains clean.

Maintenance can be triggered via:
```bash
make run-spark FILE=pipelines/test/maintenance.py ARGS="catalog_iceberg.bus_bronze.bus_way_point"
```

## 3. Usage

Most pipelines can be executed via the `Makefile`:
```bash
# Run Silver Cleaning
make run-spark FILE=pipelines/silver/silver_buswaypoint.py

# Run Gold Trip Segmentation
make run-spark FILE=pipelines/gold/trip_summary.py
```
