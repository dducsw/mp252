# Ho Chi Minh City Bus GPS Dataset

This project utilizes a large-scale bus Global Positioning System (GPS) trajectory dataset from the public bus network in Ho Chi Minh City, Vietnam.

## 1. Dataset Overview

The dataset captures real-world, streaming-like GPS messages generated by buses operating across the city. It is specifically prepared for research and educational purposes in the field of Big Data and Transportation Analytics.

- **Authors**: Khang Nguyen Duy and Nam Thoai (HPCC Lab, HCMUT, VNU-HCM).
- **Time Period**: 50 continuous days (March 20 – May 10, 2025).
- **Coverage**: 31 representative bus routes, reflecting diverse operational dynamics (length, frequency, stop density).
- **Volume**: Approximately 34GB (decompressed), containing millions of GPS messages.
- **Kaggle Link**: [Ho Chi Minh City Bus Waypoint Dataset](https://kaggle.com/datasets/e42c91f126e0df5b66879f9bfcf72d437411e34cf8557bbdbfc446616781ef9c)

## 2. Development vs. Production Data Flow

It is important to distinguish how data flows in different environments:

### Development (Dev) - Current State
- **Format**: JSON.
- **Source**: Static historical files from the Kaggle dataset.
- **Ingestion**: Simulated using `scripts/json_producer.py`, which pushes JSON records to Kafka to mimic real-time behavior.
- **Purpose**: Used for developing transformations, testing window logic, and building dashboards without needing a live connection.

### Production (Prod) - Real-world Architecture
- **Format**: **Protobuf (Protocol Buffers)**.
- **Source**: Official central data feed from the Management Centre of Public Transport (MCPT).
- **Ingestion**: In a production environment, the platform is designed to consume highly efficient binary Protobuf messages. This ensures low latency and minimal bandwidth usage for high-frequency GPS updates across the entire city fleet.
- **Integration**: The processing pipelines (Spark/Redis) are structured to be easily adaptable to the Protobuf schema by swapping the Kafka deserializer.

### Special Events & Disruptions
The dataset is unique because it captures significant real-world traffic disruptions, particularly the **April 30th military parade period (April 24–30)**, which involved extensive road closures and route detours in the city center. These events provide excellent opportunities for **Anomaly Detection** and **Event Impact Analysis**.

## 2. Data Dictionary

Each GPS record (stored in JSON format) follows the schema below:

| Feature | Datatype | Description |
| :--- | :--- | :--- |
| `vehicle` | string | Anonymized bus license plate. |
| `driver` | string | Anonymized driver identifier. |
| `speed` | float | Current speed in km/h. |
| `datetime` | int32 | Unix Timestamp (Epoch). |
| `x` | double | Latitude. |
| `y` | double | Longitude. |
| `heading` | float | Compass direction (0-360). |
| `ignition` | bool | Engine status. |
| `aircon` | bool | Air conditioning status. |
| `sos` | bool | Emergency signal status. |
| `working` | bool | Operating status (e.g., school bus mode). |

## 3. Data Processing Scripts

### 3.1 Metadata Crawling (`scripts/crawl_routes.py`)
Since the core dataset primarily contains GPS trajectories, additional metadata (Route Paths, Bus Stops, Timetables) must be enriched.
- **Source**: [buyttphcm.com.vn](https://buyttphcm.com.vn) (Official MCPT Website).
- **Usage**: Run this script to fetch the latest stop coordinates and route geometries to improve the accuracy of trip segmentation and map visualization.

### 3.2 Real-time Simulation (`scripts/json_producer.py`)
To simulate a production streaming environment, this script acts as a **Kafka Producer**.
- **Function**: Reads the historical JSON files and pushes them to a Kafka topic (`buswaypoint_json`) at a controlled rate (configurable `records_per_second`).
- **Features**: Supports **checkpointing** to resume ingestion from the last processed file/index in case of failure.
